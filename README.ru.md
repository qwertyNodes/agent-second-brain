**Персональный AI-ассистент с долговременной памятью и кривой забывания.**

Записывайте голосовые, текст и фото в Telegram -- агент сам классифицирует записи, создаст задачи в Todoist, сохранит мысли в Obsidian и отправит HTML-отчёт. Во второй версии добавлена система памяти на основе кривой Эббингауза, автоматический скоринг здоровья vault и трёхфазный pipeline обработки.

## Что нового в v2

- **Agent Memory** -- кривая забывания Эббингауза с пятью уровнями затухания: core, active, warm, cold, archive. Карточки памяти теряют вес со временем, а обращение к ним возвращает их в активный слой.
- **Vault Health** -- автоматический скоринг здоровья хранилища, генерация Maps of Content, починка битых ссылок, поиск orphan-заметок без связей.
- **3-фазный Pipeline** -- обработка разделена на CAPTURE, EXECUTE и REFLECT с передачей JSON между фазами. Каждая фаза изолирована и может быть перезапущена отдельно.
- **Правила форматирования** -- obsidian-markdown, weekly-reflection и другие правила в `.claude/rules/` для единообразия записей.
- **Конфиг памяти** -- файл `.memory-config.json` позволяет настроить скорость затухания, пороги уровней и автоматическое определение типа карточки.

## Архитектура

```
Telegram (голос/текст/фото)
    |
Telegram Bot (Node.js) --> Deepgram (голос-->текст)
    |
Claude Code (--print --dangerously-skip-permissions)
    |
+-- CAPTURE: классификация записей --> JSON
+-- EXECUTE: задачи Todoist + записи в vault --> JSON
+-- REFLECT: HTML-отчёт + обновление MEMORY
    |
+-- Todoist (задачи)
+-- Obsidian vault (мысли, CRM, цели)
+-- Telegram (HTML-отчёт)
+-- GitHub (git push)
```

Бот работает на VPS и доступен 24/7. Ежедневно в 21:00 запускается автоматическая обработка через systemd timer. Все изменения коммитятся в Git и синхронизируются с GitHub.

## Быстрый старт

### 1. Сделайте Fork

Откройте [репозиторий](https://github.com/smixs/agent-second-brain) и нажмите кнопку **Fork** в правом верхнем углу.

> **Важно:** Сделайте репозиторий приватным. Ваши цели, мысли и записи не должны быть публичными.
> Settings -> Danger Zone -> Change visibility -> Private.

### 2. Клонируйте на компьютер

```bash
git clone https://github.com/YOUR_USERNAME/agent-second-brain.git
cd agent-second-brain
```

### 3. Заполните шаблоны

Откройте и отредактируйте под себя:

- `vault/goals/0-vision-3y.md` -- видение на 3 года
- `vault/goals/1-yearly.md` -- цели на год
- `vault/goals/2-monthly.md` -- приоритеты на месяц
- `vault/goals/3-weekly.md` -- фокус на неделю (ONE Big Thing)
- `vault/.claude/skills/dbrain-processor/references/about.md` -- ваш профиль
- `vault/.claude/skills/dbrain-processor/references/classification.md` -- категории записей

> **Совет:** Скопируйте содержимое шаблона в Claude или ChatGPT и попросите заполнить. AI задаст уточняющие вопросы по одному и вернёт готовый файл. Это самый быстрый способ -- не нужно придумывать формулировки самому.

### 4. Получите API-ключи

| Токен | Где получить | Как выглядит |
|-------|-------------|--------------|
| Telegram Bot Token | [@BotFather](https://t.me/BotFather) в Telegram | `7123456789:AAH...` |
| Telegram ID | [@userinfobot](https://t.me/userinfobot) в Telegram | `123456789` |
| Deepgram API Key | [console.deepgram.com](https://console.deepgram.com/) | Длинная строка |
| Todoist API Token | [todoist.com](https://todoist.com) -> Settings -> Integrations | 40 символов |

### 5. Разверните на VPS

Подключитесь к серверу и запустите bootstrap-скрипт:

```bash
curl -fsSL https://raw.githubusercontent.com/YOUR_USERNAME/agent-second-brain/main/bootstrap.sh | bash
```

Скрипт сам скачает проект, спросит все токены, создаст `.env`, настроит автозапуск и запустит бота.

Если нужна пошаговая ручная настройка: [docs/vps-setup.md](docs/vps-setup.md)

## Структура vault

```
vault/
+-- daily/           # Ежедневные записи (транскрипты голосовых, заметки)
+-- goals/           # Видение, годовые, месячные, недельные цели
+-- business/
|   +-- crm/         # Карточки клиентов
|   +-- network/     # Профессиональные контакты
+-- projects/
|   +-- clients/     # Заметки по клиентским проектам
|   +-- leads/       # Воронка продаж
+-- thoughts/
|   +-- ideas/       # Идеи и мозговые штурмы
|   +-- learnings/   # Выученные уроки
|   +-- reflections/ # Личные рефлексии
+-- templates/       # Шаблоны карточек (CRM, daily)
+-- MOC/             # Maps of Content (авто-генерация)
+-- MEMORY.md        # Долгосрочная память (hot context)
+-- .claude/         # Конфигурация агента
    +-- skills/      # 5 скиллов
    +-- rules/       # Правила форматирования
    +-- docs/        # Справочная документация
```

Агент умеет навигироваться по vault через hub-файлы (`_index.md` в каждой папке) и wiki-links. Чем больше связей между заметками -- тем точнее контекст и лучше качество обработки.

## Как это работает

### 3-фазный pipeline

Обработка записей разделена на три изолированные фазы:

1. **CAPTURE** -- чтение `daily/*.md` за сегодня, классификация каждой записи (задача, мысль, идея, CRM, вопрос и т.д.), формирование JSON с результатами.
2. **EXECUTE** -- на основе JSON из CAPTURE создаёт задачи в Todoist, сохраняет мысли в соответствующие папки vault, обновляет CRM-карточки, добавляет wiki-links.
3. **REFLECT** -- генерирует HTML-отчёт за день, обновляет MEMORY.md, отправляет отчёт в Telegram, коммитит все изменения в Git.

Каждая фаза получает на вход JSON предыдущей и выдаёт свой JSON. Если фаза упала -- можно перезапустить только её, не теряя результаты предыдущих.

### Затухание памяти

Система использует линейную модель затухания, вдохновлённую кривой Эббингауза:

- **Скорость затухания:** rate = 0.015 в день
- **Минимальный вес:** floor = 0.1 (карточка никогда не исчезает полностью)
- **Формула:** `weight = max(floor, 1.0 - rate * days_since_last_access)`

Пять уровней по весу:

| Уровень | Диапазон веса | Описание |
|---------|---------------|----------|
| core | 0.85 -- 1.00 | Только что созданные или недавно использованные |
| active | 0.60 -- 0.84 | Регулярно используемые |
| warm | 0.35 -- 0.59 | Периодически всплывающие |
| cold | 0.15 -- 0.34 | Давно не использованные |
| archive | 0.00 -- 0.14 | Практически забытые, но не удалённые |

Обращение к карточке (recall) сбрасывает вес обратно до 1.0 -- как повторение материала в реальной жизни.

### Поиск по уровням

В зависимости от типа запроса агент ищет по разным слоям памяти:

| Режим | Уровни | Когда используется |
|-------|--------|--------------------|
| **heartbeat** | core + active | Быстрый ответ на повседневные вопросы |
| **normal** | core + active + warm | Стандартный поиск по запросу |
| **deep** | все уровни | Глубокий анализ, ретроспектива |
| **creative** | случайная выборка из cold | Генерация неожиданных связей и идей |

Режим **creative** особенно полезен: агент случайно достаёт давно забытые карточки и находит неочевидные пересечения с текущим контекстом.

### Скоринг здоровья vault

Здоровье vault оценивается по шкале от 0 до 100:

```
health = 100 - orphan_penalty - broken_penalty - density_penalty - desc_penalty
```

| Метрика | Что проверяет |
|---------|---------------|
| **orphan_penalty** | Заметки без входящих и исходящих связей |
| **broken_penalty** | Битые `[[wiki-links]]`, ведущие в никуда |
| **density_penalty** | Низкая плотность связей между заметками |
| **desc_penalty** | Отсутствие description в frontmatter (нужен для поиска) |

Скилл vault-health запускается автоматически и генерирует отчёт с конкретными рекомендациями: какие заметки связать, какие ссылки починить, где добавить описание.

## Помощь

### Бот не отвечает в Telegram

Подключитесь к серверу и проверьте статус:

```bash
sudo systemctl status d-brain-bot
```

Должно показать `active (running)`. Если нет -- перезапустите:

```bash
sudo systemctl restart d-brain-bot
```

Посмотреть логи для диагностики:

```bash
sudo journalctl -u d-brain-bot -f --no-pager -n 50
```

### Голосовые не распознаются

Проверьте Deepgram API Key в файле `.env`. Убедитесь, что ключ актуален и не истёк стартовый кредит ($200).

### Ошибки Claude

Проверьте авторизацию:

```bash
claude auth status
```

Если нужно авторизоваться заново:

```bash
claude auth login
```

### Интерактивная отладка

Самый быстрый способ разобраться с любой проблемой -- запустить Claude прямо в директории проекта:

```bash
cd ~/agent-second-brain
claude
```

Опишите проблему, и агент сам прочитает конфигурацию, проверит логи и предложит решение.
